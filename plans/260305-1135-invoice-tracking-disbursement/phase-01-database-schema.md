# Phase 1: Database Schema

## Overview

Add 4 new Prisma models: `Loan`, `Disbursement`, `Invoice`, `AppNotification`. Run migration. Update Customer model with new relation. Chain: Customer → Loan → Disbursement → Invoice.

## Key Insights

- SQLite via `better-sqlite3` adapter -- no native JSON column, use `String` + manual parse
- Existing Customer model in `@@map("customers")` -- add `disbursements Disbursement[]` relation
- `cuid()` IDs match existing pattern
- `@@index` on foreign keys and frequently queried columns (status, dueDate, readAt)

## Requirements

### Functional
- Loan belongs to Customer (1:N)
- Disbursement belongs to Loan (1:N)
- Invoice belongs to Disbursement (1:N)
- AppNotification is standalone (metadata JSON links to entities)
- Invoice uniqueness: `@@unique([invoiceNumber, supplierName])` -- duplicate = same invoice number + same supplier
- Loan fields: contract number, loan amount, interest rate, start/end date, purpose, status

### Non-functional
- Migration must be additive only (no breaking changes to existing tables)
- Indexes on query-hot columns

## Architecture

```
Customer 1--N Loan 1--N Disbursement 1--N Invoice
AppNotification (standalone, metadata JSON references invoiceId/loanId/customerId)
```

## Implementation Steps

### 1. Update `prisma/schema.prisma`

Add to existing `Customer` model:
```prisma
loans Loan[]
```

Add new models after existing ones:

```prisma
model Loan {
  id              String   @id @default(cuid())
  createdAt       DateTime @default(now())
  updatedAt       DateTime @updatedAt
  customerId      String
  customer        Customer @relation(fields: [customerId], references: [id], onDelete: Cascade)
  contractNumber  String   @unique      // Số hợp đồng vay
  loanAmount      Float                 // Hạn mức / số tiền vay
  interestRate    Float?                // Lãi suất (%)
  startDate       DateTime              // Ngày bắt đầu
  endDate         DateTime              // Ngày kết thúc
  purpose         String?               // Mục đích vay
  status          String   @default("active")  // active | completed | cancelled
  disbursements   Disbursement[]

  @@index([customerId])
  @@index([status])
  @@map("loans")
}

model Disbursement {
  id               String   @id @default(cuid())
  createdAt        DateTime @default(now())
  updatedAt        DateTime @updatedAt
  loanId           String
  loan             Loan     @relation(fields: [loanId], references: [id], onDelete: Cascade)
  amount           Float
  disbursementDate DateTime
  description      String?
  status           String   @default("active")  // active | completed | cancelled
  invoices         Invoice[]

  @@index([loanId])
  @@index([status])
  @@map("disbursements")
}

model Invoice {
  id             String       @id @default(cuid())
  createdAt      DateTime     @default(now())
  updatedAt      DateTime     @updatedAt
  disbursementId String
  disbursement   Disbursement @relation(fields: [disbursementId], references: [id], onDelete: Cascade)
  invoiceNumber  String
  supplierName   String
  amount         Float
  issueDate      DateTime
  dueDate        DateTime
  customDeadline DateTime?
  status         String       @default("pending")  // pending | paid | overdue
  notes          String?

  @@unique([invoiceNumber, supplierName])  // duplicate = same number + same supplier
  @@index([disbursementId])
  @@index([status])
  @@index([dueDate])
  @@map("invoices")
}

model AppNotification {
  id        String    @id @default(cuid())
  createdAt DateTime  @default(now())
  readAt    DateTime?
  type      String    // invoice_due_soon | invoice_overdue | duplicate_invoice
  title     String
  message   String
  metadata  String?   @default("{}")  // JSON: { invoiceId, disbursementId, customerId }

  @@index([readAt])
  @@index([type])
  @@map("app_notifications")
}
```

### 2. Generate and run migration

```bash
npx prisma migrate dev --name add-disbursement-invoice-notification
```

### 3. Verify generated client types

```bash
npx prisma generate
```

Confirm `Disbursement`, `Invoice`, `AppNotification` types available in `@prisma/client`.

## Implementation Summary

4 Prisma models added to `prisma/schema.prisma`:
- **Loan** (customerId FK to Customer, contractNumber unique, status enum)
- **Disbursement** (loanId FK, amount, disbursementDate)
- **Invoice** (disbursementId FK, invoiceNumber + supplierName unique, customDeadline optional, status pending|paid|overdue)

Indexes added on: customerId, loanId, disbursementId, status, dueDate, readAt for query optimization.
